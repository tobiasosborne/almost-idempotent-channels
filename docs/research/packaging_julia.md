## Summary recommendation

Ship the C core to Julia users via **BinaryBuilder.jl + Yggdrasil â†’
`AlmostIdempotentChannels_jll`**, with the user-facing package depending on that
JLL and (only if it calls `libflint` directly) on `FLINT_jll`. **Drop
`deps/build.jl`** â€” it is appropriate during active development of `libaic.so`
but is not acceptable for a registrable General-registry package (forces every
user to have a C toolchain + FLINT headers + LAPACK dev headers; CI rejects /
silently breaks on untested platforms). In 2025 no serious Julia package wrapping
a C lib ships a `deps/build.jl`; they ship a JLL. **Nemo.jl is the near-exact
template** (it wraps the same FLINT/arb stack).

## The JLL pipeline

A JLL is a normal Julia package, auto-generated by BinaryBuilder after a
Yggdrasil PR merges: `Artifacts.toml` (per-platform tarballs on GitHub Releases),
a platform dispatcher, and `src/wrappers/` exporting `libaic` (a basename
string), `libaic_path`, `libaic_handle`. The user package then does
`@ccall libaic.fn(...)::Ret`.

### `build_tarballs.jl` skeleton (Yggdrasil `A/AlmostIdempotentChannels/`)

```julia
using BinaryBuilder, Pkg
name    = "AlmostIdempotentChannels"     # valid Julia identifier, no dashes
version = v"0.1.0"
sources = [GitSource("https://github.com/tobiasosborne/almost-idempotent-channels.git",
                     "COMMIT_SHA")]
script = raw"""
cd $WORKSPACE/srcdir
cmake -DCMAKE_INSTALL_PREFIX=$prefix \
      -DCMAKE_TOOLCHAIN_FILE=${CMAKE_TARGET_TOOLCHAIN} \
      -DCMAKE_BUILD_TYPE=Release .
make -j${nproc}
make install
install_license LICENSE
"""
platforms = supported_platforms()        # pure C: no expand_cxxstring_abis / gfortran
products  = [LibraryProduct("libaic", :libaic)]
dependencies = [
    Dependency("FLINT_jll"),                     # libflint = arb + FLINT since v3
    Dependency("OpenBLAS32_jll"),                # 32-bit-int BLAS/LAPACK â€” see GOTCHA
    Dependency("CompilerSupportLibraries_jll"),
]
build_tarballs(ARGS, name, version, sources, script, platforms, products,
               dependencies; julia_compat="1.9")
```

This requires the CMake build from `packaging_c_core.md` (BinaryBuilder runs
`cmake â€¦ && make install` and expects a GNUInstallDirs layout:
`${prefix}/lib/libaic.so`, `${prefix}/include/aic/*.h`).

## GOTCHA #1 (the #1 from-scratch JLL bug): OpenBLAS32, not OpenBLAS

`FLINT_jll` links **`OpenBLAS32_jll`** (LP64 / 32-bit-integer BLAS, routed
through libblastrampoline). If `libaic.so` calls LAPACK directly (it does:
`zheev`, `zgesvd`, `zgees`), it **must link the same `OpenBLAS32_jll`**. Mixing
`OpenBLAS_jll` (ILP64) and `OpenBLAS32_jll` in one binary causes **silent
integer-width ABI corruption**. Verify with `ldd libaic.so` after a local build.

## GOTCHA #2: FLINT major version is ABI-breaking, share one libflint

arb is **inside** FLINT since 2.9/3.0 â€” there is no `Arb_jll`; `FLINT_jll`
(v301.x = FLINT 3.x) provides a single `libflint.so` with all `arb_*`/`acb_*`/
`acb_mat_*`. Because all JLLs share one Julia artifact store, exactly **one**
`libflint.so` is loaded per process â€” `libaic.so` and any Julia code calling
`libflint` share it (same symbols, same `arb_t` layout). `[compat]` for
`FLINT_jll` must pin the major (`"301"`); FLINT 2.xâ†”3.x layouts differ. Build the
C core against the same FLINT major the JLL pins. Build the C core thread-safe
(`FLINT_THREADED` / `--enable-thread-safe`) since the Julia GC spawns threads.

Every `[deps]` entry needs a `[compat]` upper bound (registry requirement). Only

inspects certified output balls); otherwise it is transitive through the JLL.

## ccall idioms (`@ccall`, opaque handles + finalizers)

```julia
using AlmostIdempotentChannels_jll: libaic
mutable struct AICChannel end                 # opaque C type sentinel
mutable struct AICHandle
    ptr::Ptr{AICChannel}
    function AICHandle(n::Int)
        p = @ccall libaic.aic_channel_alloc(n::Cint)::Ptr{AICChannel}
        p == C_NULL && throw(OutOfMemoryError())
        h = new(p)
        finalizer(o -> @ccall(libaic.aic_channel_free(o.ptr::Ptr{AICChannel})::Cvoid), h)
        return h
    end
end
Base.cconvert(::Type{Ptr{AICChannel}}, h::AICHandle) = h
Base.unsafe_convert(::Type{Ptr{AICChannel}}, h::AICHandle) = h.ptr
```

- Prefer `@ccall` (Julia 1.9+) over the 4-arg `ccall`.
- Handles must be `mutable` for `finalizer`. Use `GC.@preserve` around any raw
  `Ptr` extraction used across non-ccall Julia ops. No I/O inside finalizers.
- `Ref{T}` for Julia-owned in/out; `Ptr{T}` for C-owned. For `acb_mat_t` (a
  fixed-size C struct) wrap as `Ptr{Cvoid}` in a finalized handle.

## Exemplars

- **Nemo.jl** â€” the primary template: wraps FLINT (arb inside), depends on
  `FLINT_jll`+`GMP_jll`+`MPFR_jll`, `src/` mirrors the C lib, no `deps/build.jl`,
  compat pins FLINT major. OSCAR (Nemo+Hecke+Oscar) shares one `libflint`.
- **Arblib.jl** â€” thin `arb_t`/`acb_t`/`acb_mat_t` ccall wrapper; naming
  conventions (strip prefix, `!` for mutating, precision kwargs).
- **HDF5.jl** â€” JLL + `Preferences.jl` override for a system lib
  (`@load_preference("libaic", â€¦_jll.libaic)` lets users point at a custom build
  via `LocalPreferences.toml`).
- **Yggdrasil `F/FastTransforms/build_tarballs.jl`** â€” closest structural analog
  (declares `OpenBLAS_jll`/`MPFR_jll`/`FFTW_jll` Dependency vs BuildDependency).
